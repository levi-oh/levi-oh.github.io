# State

在 RN 中 state 是一个很重要的概念，通常在一个动态页面上会存在多个可以动态改变的值，当你改变其中一个数据的时候，一般情况下需要刷新整个页面，那就会很麻烦，为了实现局部刷新，RN 引入了 state 的概念，state 可以看成某一组件的专属变量，他不能被直接赋值，在旧版的 RN 中需要调用 setState 函数才能改变 state 的值，现在我发现官网文档上的新版本中已经采用了一种更直观的方式来定义 state

[<变量名>, <set函数>] = useState(<初始值>)

这样定义之后，只要调用后面的 set函数 就能改变变量的值。
定义一个 num 放入 Text ，然后设置一个 Button 去调用 setNum ，每次给 num + 1

查看完整代码 
import React, {useState} from 'react';
import {Text, View, StyleSheet, Button} from 'react-native';
function App() {
  const name = 'Levi';
  const [num, setNum] = useState(0);
  return (
    <View style={styles.container}>
      <View style={styles.box}>
        <Text style={styles.title}>Hello, {name} !🎉</Text>
      </View>
      <View style={styles.box}>
        <Text style={styles.title}>{num}</Text>
      </View>
      <Button title="Add" onPress={() => setNum(num + 1)} />
    </View>
  );
}

export default App;

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    margin: 60,
    backgroundColor: '#ff9090',
  },
  box: {
    margin: 10,
    padding: 10,
    backgroundColor: '#f7c67e',
  },
  title: {
    fontSize: 30,
  },
});


这样就实现了一个简单计数器

# 事件处理

下面来获取一个文本输入，用 TextInput 来获取用户输入，去改变变量 name 的值，所以， name 最好也要像上面的 num 那样去定义成 state

查看完整代码 
import React, {useState} from 'react';
import {Text, View, StyleSheet, Button, TextInput} from 'react-native';
function App() {
  const [name, setName] = useState('Levi');
  const [num, setNum] = useState(0);
  return (
    <View style={styles.container}>
      <View style={styles.box}>
        <Text style={styles.title}>Hello, {name} !🎉</Text>
      </View>
      <View style={styles.box}>
        <Text style={styles.title}>{num}</Text>
      </View>
      <Button title="Add" onPress={() => setNum(num + 1)} />
      <TextInput
        placeholder="Tell me your name!"
        onChangeText={(text) => setName(text)}
        defaultValue={''}
      />
    </View>
  );
}

export default App;

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    margin: 60,
    backgroundColor: '#ff9090',
  },
  box: {
    margin: 10,
    padding: 10,
    backgroundColor: '#f7c67e',
  },
  title: {
    fontSize: 30,
  },
});

TIP

如果你想让用户点击输入栏的时候弹出数字键盘，那么可以把TextInput 的 keyboardType 属性设为 "number-pad"

更多参数请见 官方组件文档 (opens new window)

# 滚动视图

由于每台移动设备屏幕尺寸大小不一，为了尽可能的兼容大部分设备，我们要引入 ScrollView 滚动视图，他可以允许页面上的元素溢出屏幕，这时就会出现滚动条

ScrollView

ScrollView 是一个通用的可滚动的容器，你可以在其中放入多个组件和视图，而且这些组件并不需要是同类型的。ScrollView 不仅可以垂直滚动，还能水平滚动（通过 horizontal 属性来设置）。

来自 RN官方文档 (opens new window)

# 第三方组件

下面我想实现一个通过摄像头获取图片，然后上传的功能，像这种组件官方是没有的，但是与官方合作的一个集成开发环境 Expo (opens new window) 里面有提供，但是在国内不是很推荐使用那个快速框架，对网络质量要求比较高，所以在 Github 上先找一圈，找到两个备选方案

  • react-native-image-picker (opens new window)
  • react-native-image-crop-picker (opens new window)
  • react-native-image-resizer「图片压缩-留着备用」 (opens new window)
    看了两个组件的介绍，应该都能满足我的需求，这种情况下我一般会优先选则权限要求最少的，对项目影响程度最小的（尽量不要修改 Android 文件夹下的内容） ，更新时间比较近的，Star⭐ 数比较多的
    所以我选择 react-native-image-picker ，先根据说明文档里的提示安装组件（一般安装组件时会中断正在调试的程序）
yarn add react-native-image-picker


这样就算安装成功了，安装完成后要重新 启动项目
关于各类第三方组件的使用，很多网上的教程都太老了，建议自己啃官方仓库的说明文档，有的会给出 demo

  • 进入他的 example 看一看，就只要用 Button 调用他的 launchCamera 方法，然后获得一个图片地址的回调，中间可以自定义各种参数，看着应该挺易用的
  • 把 Demo 里的代码 copy 进自己的项目，稍加修改就能启动 项目启动后发现开不了摄像头，应该是软件权限没给，进入 android\app\src\main\AndroidManifest.xml 文件添加以下权限
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW"/>

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-feature android:name="android.hardware.camera" />
<uses-feature android:name="android.hardware.camera.autofocus" />

之后发现还是不行，图库读取可以了，但是摄像头还是无法启动， 在 Github 上翻了一圈 issues 终于找到了解决方案，还要在AndroidManifest.xml文件的<application> tag下新增

android:requestLegacyExternalStorage="true"
