通过smb协议实现全设备文件互通

Levi-ww 2020-12-11 samba路由器

smb协议

SMB（Server Message Block）通信协议是微软（Microsoft）和英特尔(Intel)在1987年制定的协议，主要是作为Microsoft网络的通讯协议。SMB 是在会话层（session layer）和表示层 (opens new window)（presentation layer）以及小部分应用层 (opens new window)（application layer）的协议。

SMB使用了NetBIOS的应用程序接口 (opens new window) （Application Program Interface，简称API）。另外，它是一个开放性的协议，允许了协议扩展——使得它变得更大而且复杂；大约有65个最上层的作业，而每个作业都超过120个函数，甚至Windows NT也没有全部支持到，最近微软又把 SMB 改名为 CIFS（Common Internet File System），并且加入了许多新的特色。

百度百科 (opens new window)

最近收获了一枚劣质U盘，其特性十分让人恼火，表现为时不时出现无法被识别的情况（USB2.0口），但是在USB3.0接口上表现又十分优秀，本来想用它来当临时文件中转站，但是他在小米笔记本上表现实在是不佳（能否被识别全靠运气），为了让他能继续为我的设备们发光发热，最终决定把它发派到宿舍路由器上做外接存储介质。

宿舍的路由器型号是 小米路由器3G ，在后面的USB3.0口接上U盘后，进入路由器管理页开启 samba 服务，然后为所有设备打开全盘访问权限

然后就可以用各个设备去访问路由器上的文件了，再也不用在QQ文件传输助手里麻烦地中转各种文件了🤪

# 各个系统接入方式

  • macOS

    • 打开 Finder ，⌘+k ，连接服务器，输入smb://192.168.31.1

    • 连接身份选「客人」，然后就可以成功连入了

  • Windows

    • 打开资源管理器，在地址栏输入//192.168.31.1，也可以进入路由器文件夹

    • 右键将该位置保存为网络驱动器，就能随时访问了

  • Android & iOS & iPadOS

    • 都可以通过「小米WiFi」app来访问文件
